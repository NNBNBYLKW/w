# v1.1 Batch 3：Docs 与 bootstrap 与 packaging 收口（执行版）

## 1. 文档目的

本文档用于记录 v1.1 Batch 3 的当前收口内容。

本批次只处理：

- canonical/current-state doc clarification
- concise operational entrypoint clarification
- bootstrap / migration reality clarification
- packaging / build verification wording clarification

它不是：

- 新功能计划
- migration redesign 方案
- packaging automation 方案
- CI / installer 改造说明

## 2. 当前收口内容

本批次当前已收口：

- `README.md` 作为 concise operational entrypoint
- current-state canonical doc set 的入口说明
- backend 当前 baseline-SQL bootstrap reality 的明确说明
- 当前 validation wording 的保守表达：
  - backend tests when run
  - frontend build
  - desktop build
  - manual verification

## 3. 为什么要做这次收口

本批次的目的不是扩 scope，而是减少以下理解成本：

- 新协作者不知道先看哪份文档
- startup/build/run instructions 分散在多个位置
- 当前 bootstrap reality 容易被误读为更完整 migration system
- verification wording 容易被误读为已经存在 packaging automation 或 automated release gates

## 4. 当前边界

本批次没有：

- 修改任何 backend / frontend / desktop 实现
- 修改任何构建脚本、CI workflow、migration SQL 或 packaging script
- 引入新的 release process、installer flow 或 automation system

本批次只是对当前仓库状态做文档层面的 clarify / record。

# Windows 本地资产管理工作台 低保真线框文案 v1

## 1. 文档目的

本文件用于将 PRD v1 进一步翻译为**逐页线框说明**。
目标不是输出视觉稿，而是明确：
- 页面从左到右、从上到下的结构
- 每个区域放什么
- 用户第一眼先看到什么
- 页面里的主操作和次操作分别是什么
- 空状态、加载状态、无结果状态如何落位

本文件适合作为：
- 低保真原型输入
- 静态页面搭建说明
- 前端占位实现依据

---

## 2. 全局线框基线

### 2.1 全局布局骨架
所有主要页面优先共享同一套骨架：

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部全局栏：页面标题 / 全局搜索 / 当前筛选提示 / 视图切换 / 排序 │
├──────────────┬───────────────────────────────┬──────────────┤
│ 左侧主导航   │ 中央主内容区                   │ 右侧详情侧栏 │
│              │                               │（按需展开）  │
└──────────────┴───────────────────────────────┴──────────────┘
```

### 2.2 左侧主导航
固定内容建议：
- 首页
- 全部文件
- 素材库
- 标签 / 集合
- 最近导入
- 设置
- （P1）游戏库 / 电子书库 / 软件库

### 2.3 顶部全局栏
固定元素建议：
- 当前页面标题
- 全局搜索框
- 当前页面相关的轻量控制项（排序 / 视图切换 / 清空筛选）

### 2.4 右侧详情侧栏
统一原则：
- 默认收起或占位
- 单击内容项后展开
- 承担查看详情、标签操作、颜色标签操作、打开文件、打开目录等动作

## 3. 页面 1：首次引导 / 扫描源配置页

### 3.1 页面目标
帮助用户完成第一次入库，让产品从“空壳”进入“有内容可用”的状态。

### 3.2 线框结构

```text
┌──────────────────────────────────────────────────────────┐
│ 页面标题：开始使用                                        │
│ 简短说明：选择要管理的本地目录，系统不会移动原文件          │
├──────────────────────────────────────────────────────────┤
│ 主卡片：已添加扫描源列表                                  │
│  - D:\Pictures                                           │
│  - E:\Games                                              │
│  - F:\Books                                              │
│  [启用/禁用] [删除]                                       │
├──────────────────────────────────────────────────────────┤
│ 操作区                                                    │
│ [添加目录] [添加盘符] [忽略目录设置]                      │
├──────────────────────────────────────────────────────────┤
│ 扫描区                                                    │
│ [开始扫描] [重新扫描]                                     │
│ 当前状态：未开始 / 扫描中 / 已完成 / 失败                  │
└──────────────────────────────────────────────────────────┘
```

### 3.3 视觉优先级
1. 添加扫描源
2. 已添加扫描源列表
3. 开始扫描按钮
4. 辅助说明

### 3.4 主操作
- 添加扫描源
- 删除扫描源
- 开始扫描

### 3.5 次操作
- 忽略目录设置
- 启用 / 禁用扫描源

### 3.6 空状态文案建议
- 标题：还没有添加任何扫描源
- 说明：先选择一个或多个本地目录，系统会为它们建立索引和库视图
- 主按钮：添加目录

### 3.7 加载 / 处理状态
- 扫描中时，页面下半部分出现进度区或扫描状态区
- 第一版不必做非常复杂的进度图，只要有明确状态即可

## 4. 页面 2：首页 / 入库概览页

### 4.1 页面目标
作为进入产品后的方向盘，而不是复杂分析仪表盘。

### 4.2 线框结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部：页面标题“首页”         [全局搜索框]                   │
├──────────────────────────────────────────────────────────────┤
│ 快捷入口卡片区                                                │
│ [搜索内容] [进入素材库] [查看全部文件] [最近导入]             │
├──────────────────────────────────────────────────────────────┤
│ 最近导入摘要                                                  │
│  最近新增的 6~12 个文件/素材                                  │
│  [查看全部最近导入]                                           │
├──────────────────────────────────────────────────────────────┤
│ 标签摘要区               │ 扫描源摘要区                      │
│ 常用标签 / 无标签引导     │ 扫描源数量 / 最近扫描状态          │
└──────────────────────────────────────────────────────────────┘
```

### 4.3 视觉优先级
1. 快捷入口卡片
2. 全局搜索框
3. 最近导入摘要
4. 标签摘要 / 扫描源摘要

### 4.4 主操作
- 进入搜索页
- 进入素材库
- 进入最近导入

### 4.5 次操作
- 查看扫描源状态
- 查看常用标签

### 4.6 空状态
如果还未入库：
- 页面中央显示“还没有内容进入系统”
- 主按钮：去添加扫描源

## 5. 页面 3：全局搜索结果页

### 5.1 页面目标
承载“比 Windows 更快找到内容”的第一核心价值。

### 5.2 线框结构

```text
┌────────────────────────────────────────────────────────────────────┐
│ 顶部：页面标题“搜索”    [搜索框________________] [排序] [视图切换] │
├────────────────────────────────────────────────────────────────────┤
│ 当前筛选胶囊： [图片] [蓝色标签] [赛博朋克] [清空全部]             │
├───────────────┬──────────────────────────────────────┬────────────┤
│ 左侧筛选栏    │ 中央结果区                            │ 右侧详情栏 │
│ - 类型        │ 结果总数：128                         │ 未选中占位 │
│ - 标签        │ ┌──────────────────────────────────┐ │ 或文件详情 │
│ - 颜色标签    │ │ 列表模式 / 网格模式结果展示       │ │            │
│ - 路径/来源   │ │ 文件名 / 缩略图 / 路径 / 标签提示 │ │            │
│ - 时间        │ └──────────────────────────────────┘ │            │
└───────────────┴──────────────────────────────────────┴────────────┘
```

### 5.3 视觉优先级
1. 搜索框
2. 结果区
3. 筛选器
4. 详情侧栏

### 5.4 主操作
- 输入 / 修改搜索词
- 点击筛选条件
- 单击某项查看详情
- 双击某项打开文件

### 5.5 次操作
- 切换排序
- 切换列表 / 网格
- 清空筛选

### 5.6 无结果状态
中央结果区替换为：
- 标题：没有找到匹配内容
- 说明：当前关键词或筛选条件可能过严
- 操作：清空全部筛选 / 返回全部内容

### 5.7 加载状态
- 结果区显示列表骨架或卡片骨架
- 详情侧栏若有选中项则显示局部骨架

## 6. 页面 4：素材库页

### 6.1 页面目标
作为第一版差异化最强的主页面，体现“像内容平台一样浏览本地素材”。

### 6.2 线框结构

```text
┌────────────────────────────────────────────────────────────────────┐
│ 顶部：页面标题“素材库” [图片/视频/全部] [排序] [密度切换] [搜索框] │
├────────────────────────────────────────────────────────────────────┤
│ 当前筛选胶囊： [图片] [红色标签] [盔甲] [清空全部]                 │
├───────────────┬──────────────────────────────────────┬────────────┤
│ 左侧筛选栏    │ 中央视觉内容区                        │ 右侧详情栏 │
│ - 标签        │ ┌────┐ ┌────┐ ┌────┐ ┌────┐         │ 未选中占位 │
│ - 颜色标签    │ │图/视│ │图/视│ │图/视│ │图/视│ ...  │ 或素材详情 │
│ - 类型        │ └────┘ └────┘ └────┘ └────┘         │            │
│ - 时间        │ 文件名弱化、缩略图优先                │            │
└───────────────┴──────────────────────────────────────┴────────────┘
```

### 6.3 视觉优先级
1. 素材卡片流
2. 筛选器
3. 顶部范围切换与排序
4. 详情侧栏

### 6.4 主操作
- 浏览素材
- 单击素材查看详情
- 双击素材打开文件
- 设置标签 / 颜色标签

### 6.5 次操作
- 切换图片 / 视频 / 全部
- 切换网格密度
- 调整排序

### 6.6 空状态
#### 素材库整体为空
- 标题：素材库还没有内容
- 说明：请先添加图片或视频所在目录并完成扫描
- 主按钮：去添加扫描源

#### 当前筛选下为空
- 标题：当前筛选条件下没有素材
- 主按钮：清空筛选
- 次说明：可以返回全部素材继续浏览

### 6.7 加载状态
- 中央区域显示 MediaCard 骨架
- 保留现有筛选器和顶部工具栏，避免页面跳变

```text
├──────────────────────────────┤

├──────────────────────────────┤

│ ○红 ○黄 ○绿 ○蓝 ○紫 ○无      │

├──────────────────────────────┤
│ 动作区                        │

└──────────────────────────────┘
```

### 7.3 视觉优先级
1. 预览区
2. 标签区
3. 颜色标签区
4. 打开动作
5. 元数据区

### 7.4 主操作
- 添加标签
- 设置颜色标签
- 打开文件
- 打开所在目录

### 7.5 占位状态
未选中任何内容时：
- 标题：未选择任何项目
- 说明：在左侧或中央区域单击一个文件，即可在这里查看详情和整理标签

### 7.6 加载状态
- 预览区、标签区、元数据区分别显示骨架

### 7.7 错误状态
- 标题：无法读取详情
- 按钮：重试

## 12. P1 页面占位线框建议

### 12.1 游戏库页
结构参考素材库，但卡片更偏封面 / 图标，侧栏中加入“启动”动作。

### 12.2 电子书库页

## 13. 关键状态落位规则

### 13.1 空状态落位
- 页面整体无内容：放在中央主内容区中间
- 某局部区块无内容：放在该区块内部，不遮挡整个页面

### 13.2 加载状态落位
- 列表页：列表骨架
- 卡片页：卡片骨架
- 详情侧栏：局部骨架
- 不建议整页只显示一个大转圈

### 13.3 无结果状态落位
- 出现在结果区中央
- 顶部仍保留搜索词和筛选器，便于用户调整

### 13.4 错误状态落位
- 在对应失败区域就地提示
- 提供重试按钮

---

## 14. 低保真原型绘制顺序建议

建议先画以下 6 张：
1. 全局布局骨架
2. 扫描源配置页
3. 全局搜索结果页
4. 素材库页
5. 统一详情侧栏
6. 最近导入页

然后再补：
7. 首页
8. 全部文件页
9. 标签页
10. 设置页

原因：
- 前 6 张已经能验证第一版主链
- 后 4 张更多是补全入口与体系完整性

---

## 15. 当前结论

这份低保真线框文案的核心目标，不是把所有页面画复杂，而是优先验证：

> **扫描进来 → 能搜到 → 能看清 → 能打标签 → 能再找回 → 能打开原文件**

只要这条主链在线框层已经清晰，后续视觉设计和实现就不会偏。

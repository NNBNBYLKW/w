# 正式版 v1 边界定义与不纳入范围事项

## 1. 文档目的

本文档用于在当前 Phase 2 收口后，正式冻结“正式版 v1”的产品边界。

它要回答的核心问题是：

1. 当前版本如果被定义为 `v1`，它到底包含什么
2. 哪些能力虽然以后可能会做，但**不属于 v1 的交付承诺**
3. 哪些常见误解需要提前挡住，避免后续开发把 v1 边界再次拉宽
4. 后续 Phase 3/Phase 4 的方向，不应反向污染当前 v1 定义

本文档不是继续扩功能，而是：

> **把“正式版 v1 是什么、不是什麽”写清楚。**

---

## 2. v1 的总定义

当前建议将正式版 v1 定义为：

> **一个面向本地文件资产的基础工作台，支持 source 管理、扫描与索引、检索与浏览、基础元数据与图像预览、标签与颜色标签组织、以及可保存的 collections 入口。**

这个定义有三个关键点：

### 2.1 v1 是“基础工作台”
也就是说：
- 它是一个完整可用的本地资产管理入口
- 它有真实的主链和组织能力
- 它不是只会扫文件的脚手架，也不是只会搜索的 demo

### 2.2 v1 面向“本地文件资产”
它当前处理的核心对象仍然是：
- indexed files
- 本地 source
- 文件级 metadata / thumbnail / preview / tags / color tags

而不是：
- 云端对象
- 多端同步对象
- AI 生成对象中心
- 游戏/书籍/软件的完整垂类平台对象

### 2.3 v1 是“基础版”，不是“终局版”
它意味着：
- 当前能力已经足够正式交付和试用
- 但并不等于未来路线图里的所有目标都要在 v1 完成

---

## 3. v1 明确包含的能力

以下能力属于正式版 v1 的明确范围。

## 3.1 Source 与扫描链
v1 包含：
- source 创建
- source 更新
- source 删除
- source root 校验
- source overlap/duplicate 防护
- 手动触发 scan
- delete-sync
- 同 source active scan 冲突保护
- 最近扫描失败的最小反馈

这意味着：
- v1 不是静态文件列表工具
- 它具备真实的 source -> scan -> indexed files 主链

---

## 3.2 文件检索与浏览
v1 包含以下检索/浏览面：

- Search
- Files
- Media Library
- Recent Imports
- Tags
- Collections

其中：

### Search
支持：
- 文本搜索
- file_type filter
- tag filter
- color tag filter
- 分页、排序

### Files
支持：
- flat indexed-files listing
- source browse
- exact-directory browse
- tag filter
- color tag filter
- 分页、排序

### Media Library
支持：
- active indexed media listing
- image thumbnail
- image preview through shared details panel

### Recent Imports
支持：
- 最近导入文件列表
- 时间范围切换
- 排序、分页

### Tags
支持：
- 按普通标签找回文件

### Collections
支持：
- 保存最小结构化条件
- 实时结果列表
- 长期可复用入口

---

## 3.3 组织层能力
v1 包含：
- 普通标签 attach / remove
- 颜色标签 set / clear
- Search / Files 中的 tag/color 检索增强
- Collections 作为长期入口

这意味着：
- v1 已经不只是“扫了之后找文件”
- 它已经支持“整理 -> 再找回”的最小闭环

---

## 3.4 文件内容增强层
v1 包含：
- Phase 2A metadata baseline
- image `width` / `height`
- image thumbnail
- image preview

这意味着：
- v1 已经不是纯文本列表工作台
- 至少对 image 文件，已经具备最小但真实的内容可见性

---

## 3.5 详情与动作
v1 包含：
- shared `DetailsPanelFeature`
- file details
- metadata 展示
- tags / color tags 展示与操作
- open file
- open containing folder

这意味着：
- 主要页面都复用同一个详情与动作入口
- v1 的动作面没有分裂成多个 page-local detail systems

---

## 3.6 主入口与页面骨架
v1 包含：
- Home
- Settings
- Tags
- Collections
- Search
- Files
- Media Library
- Recent Imports

这些页面在 v1 中都不是占位页，而是已具备真实作用。

---

## 4. v1 明确不包含的能力

以下能力即使未来可能会做，也**不属于 v1 承诺范围**。

## 4.1 不包含完整任务平台
v1 不包含：
- `/tasks` 页面
- `/tasks` API
- 任务历史列表
- retry 系统
- scan queue
- worker orchestration
- runtime dashboard

### 说明
当前 v1 只包含：
- inline scan 模型
- same-source conflict protection
- source-level failure feedback

这不等于“完整任务平台已完成”。

---

## 4.2 不包含完整 preview/media 系统
v1 不包含：
- video poster extraction
- video hover/play
- document preview
- preview URL 系统
- 富媒体播放器
- thumbnail DB 子系统

### 说明
当前 v1 的 preview 能力仅限：
- image thumbnail
- image preview

---

## 4.3 不包含更完整 metadata / content understanding
v1 不包含：
- video metadata 正式激活
- document page count 正式激活
- title / author / series / codec info
- OCR
- embedding / semantic indexing
- AI content understanding

### 说明
当前 metadata baseline 只是第一层最小可用内容特征层。

---

## 4.4 不包含高级组织系统
v1 不包含：
- smart rules / auto collections
- advanced query builder
- OR / NOT / nested boolean logic
- multi-tag arrays
- batch organization actions
- tag rename / merge / delete 扩张

### 说明
当前组织层的正式边界是：
- tags
- color tags
- Search/Files retrieval loop
- smart collections baseline

而不是“高级自动化组织平台”。

---

## 4.5 不包含垂类库系统
v1 不包含：
- 游戏库
- 电子书库
- 软件库
- 对这些垂类对象的专属状态/封面/详情页体系

### 说明
这些仍然属于后续可能进入的 Phase 3 或更后阶段，不应拿来作为 v1 的隐性要求。

---

## 4.6 不包含多端/云端能力
v1 不包含：
- 云同步
- 多端共享
- 账户体系
- 远程库
- collection sharing
- collaboration

### 说明
当前产品仍然明确是 **local-first 本地资产工作台**。

---

## 5. 常见误解与纠偏

为了防止后续规划或沟通中不断拉宽 v1，以下误解需要提前纠正。

## 5.1 “有 Collections，就应该有规则引擎”
错误。

当前 v1 的 Collections 只是：
- 保存最小结构化条件
- 再次进入实时结果视图

它不是：
- 自动执行规则
- 周期任务系统
- 智能整理引擎

---

## 5.2 “有 metadata 和 thumbnail，就应该支持所有媒体类型”
错误。

当前 v1 只正式支持：
- image metadata
- image thumbnail / preview

这并不自动意味着：
- video poster
- video metadata
- document preview

都已经属于 v1。

---

## 5.3 “有 scan/runtime hardening，就应该有 task center”
错误。

当前 v1 只补了：
- same-source active scan conflict
- `last_scan_error_message`
- source surface 的最小运行时表达

它并不是完整任务平台。

---

## 5.4 “有 Search / Files filter，就应该支持任意复杂组合查询”
错误。

当前 v1 的过滤语义仍然刻意保持简单：
- 单值 tag
- 单值 color tag
- 纯 `AND`
- 不支持多组逻辑

这不是缺陷，而是 v1 的边界选择。

---

## 5.5 “正式版 v1 就必须包含游戏库 / 书库 / 软件库”
错误。

如果 v1 的定义是“完整垂类内容平台”，那当前当然不够。  
但当前更合理的 v1 定义是：

> **本地资产工作台基础版**

按这个定义，当前边界是成立的。

---

## 6. v1 的推荐官方口径

如果你后续需要对外或对自己定义版本，当前最推荐的表达是：

> **v1 是一个 local-first 的本地资产工作台基础版，支持 source 管理、扫描索引、搜索与浏览、基础 metadata、图像缩略图与预览、标签/颜色标签组织，以及 Collections 长期入口。**

这个表述有几个好处：
- 它和当前真实代码状态一致
- 它不会把未来 Phase 3/4 的能力提前许诺进来
- 它也不会把当前版本说得太弱

---

## 7. v1 的推荐发布边界

当前建议将 v1 的发布边界冻结为：

### 必须成立的能力
- Source onboarding + scan
- delete-sync
- Search / Files / Media / Recent / Tags / Collections
- shared DetailsPanel
- tags / color tags
- metadata baseline
- image thumbnail / preview
- source runtime hardening
- desktop open actions

### 明确不要求 v1 交付的能力
- async runtime
- 游戏/书籍/软件库
- 高级 query builder
- smart rules
- richer video/document preview
- AI / semantic layer

---

## 8. 当前最合理的下一步

既然 v1 边界已经可以冻结，当前最合理的下一步就不是“继续横向补功能”，而是：

> **进入 Release / Polish 阶段。**

推荐做法：
- 统一文案
- 清理 UI 小瑕疵
- 整理 known issues
- 补充正式版手工验收路径
- 做一次 release freeze

而不是立刻进入：
- 新垂类库
- 新任务系统
- AI 层

---

## 9. 最终结论

### 正式版 v1 的定位
> **本地资产工作台基础版**

### v1 已明确包含
- source / scan / indexed retrieval 主链
- metadata baseline
- image thumbnail / preview
- tags / color tags
- Search / Files retrieval loop
- smart collections baseline
- shared details & desktop actions

### v1 明确不包含
- 完整任务平台
- 完整多媒体预览系统
- 高级组织系统
- 垂类库平台
- AI / semantic / automation

### 当前推荐动作
> **停止继续拉宽 v1，进入 Release / Polish 阶段。**

---

## 10. 后续建议文档

在此之后，最建议继续补的文档是：

1. `Release / Polish 阶段任务清单`
2. `正式版 v1 已知问题与非阻塞缺陷清单`

这样你就能正式把当前项目从“持续开发”切换为“准备定版”。

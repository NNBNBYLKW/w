# 测试版发布准备

## 1. 当前 beta 发布目标

本文件说明 `Workbench Beta 0.2.0` 的当前交付形态、验证边界和已知限制。

当前推荐交付物是 **一体化 Windows beta 包**：用户安装后双击启动，不需要手动启动 Python backend，也不需要自行配置 FFmpeg。

这仍然是 beta 发布准备文档，不是正式商业发布说明，也不是新功能规划。

## 2. 当前 beta 包包含什么

当前一体化 Windows beta 包包含：

- Electron 桌面壳
- 已构建的 frontend 静态资源
- PyInstaller `onedir` 打包的 backend executable
- 随包携带的 `ffmpeg.exe`
- 随 backend 打包的 `pypdfium2` / PDFium runtime，用于 PDF 第一页缩略图
- 桌面 bridge：选择文件夹、打开文件、打开所在文件夹、窗口控制

当前 beta 包不包含：

- 自动更新系统
- 后台服务安装或系统级自启动
- 复杂安装向导
- 正式发布级应用图标 / 签名收口
- SQLite 初始用户数据

旧的技术 beta 包已归档到：

```text
apps/desktop/release/archive/technical-beta-0.1.0-manual-backend-20260425/
```

该旧包需要用户手动启动 backend，仅作为回退参考，不再作为推荐交付形态。

## 3. 安装 / 启动前提

面向非技术用户的一体化包预期行为：

- 安装后双击 `Workbench Beta`
- Electron 自动启动本地 backend
- backend ready 后进入 frontend
- 应用退出时关闭本次启动的 backend 子进程
- 视频缩略图与 DetailsPanel 6 帧预览默认使用随包携带的 FFmpeg
- PDF 第一页缩略图默认使用随 backend 包携带的 pypdfium2 / PDFium
- Documents 用户侧命名已替代 Books；内部 `/library/books` route 与 `books` placement 仍作为兼容层
- Games 预览复用统一 thumbnail pipeline；无真实 thumbnail 的 game/archive/rom/iso/shortcut 条目显示 fallback
- Settings 语言切换会即时刷新 Sidebar / Shell / 当前页面 / DetailsPanel
- Home / Search / Files / Recent 的深色模式颜色状态已完成当前验收

packaged mode 的本地 backend 默认运行在：

```text
http://127.0.0.1:8765
```

开发模式仍可直接运行 backend：

```powershell
cd apps/backend
..\..\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

开发验收更推荐使用仓库根目录的 `start-dev.bat` 或：

```powershell
.\start-dev.ps1 -StartDesktop
```

启动前需要确认 `8000` 没有残留旧 backend。旧的 `uvicorn --reload` reloader / server 子进程如果没有通过 `Ctrl+C` 正常停止，可能继续占用端口，让手工验收命中旧代码。

开发模式启动后应检查：

```powershell
$base = "http://127.0.0.1:8000"

Invoke-RestMethod "$base/debug/runtime" -TimeoutSec 5 | ConvertTo-Json -Depth 8
Invoke-RestMethod "$base/debug/thumbnails/warmup" -TimeoutSec 5 | ConvertTo-Json -Depth 8
```

异常判断：

- `sys_executable` 是 `C:\Python314\python.exe`：没有使用项目 `.venv`，或访问到了旧进程。
- `process_id` / `process_start_time` 缺失：当前 backend 可能是旧代码。
- `pdf_render_mode=subprocess-v1` 缺失：thumbnail warmup 不是当前实现。

## 4. 数据目录

一体化包不把正式运行数据写回仓库目录。

packaged mode 下运行数据写入 Electron `userData` 目录，例如：

```text
%APPDATA%/Workbench Beta/backend-data/
```

其中包含：

- `workbench.db`
- `thumbnails/`
- `thumbnails/video/`
- `thumbnails/video_preview/`
- `thumbnails/pdf/`

开发模式仍默认使用：

```text
apps/backend/data/
```

## 5. 当前打包命令

```powershell
cd apps/desktop
npm run package:win
```

该命令会依次完成：

- frontend production build
- backend PyInstaller build
- bundled FFmpeg resource prepare
- desktop TypeScript build
- electron-builder Windows NSIS package

Windows 一体化包输出目录：

```text
apps/desktop/release/integrated/
```

当前安装包名：

```text
Workbench Beta Integrated Setup 0.1.0.exe
```

## 6. 建议 smoke test

发布前至少检查：

- 不手动启动 Python backend，直接启动 `win-unpacked/Workbench Beta.exe`
- `/health` 在 `http://127.0.0.1:8765` 可访问
- 无 saved source 时进入 `Onboarding`
- 有 saved source 时进入 `Home`
- 左侧导航可进入 Search / Library / Files / Recent / Media / Documents / Games / Software / Tools / Tags / Collections / Settings
- 单击条目能刷新 shared details
- 双击条目或 details open actions 在 Electron 中不崩溃
- tags / color tags / favorite / rating 可写入与回显
- Recent batch organize 可进入、操作后可再找回
- Collections / Tags / Recent family 的回流入口可用
- Library Objects tab 可浏览，Pending tab 可扫描候选并生成计划
- Library Plans tab 可查看计划详情，Preflight / Execute / Logs 流程可用
- Settings 语言切换可用
- Settings Light / Dark 主题切换可用，并在刷新或重启后保持
- Search / Documents / Software / Media / Games 的 Details / Icons 视图切换可用
- 图片缩略图可用
- 视频缩略图可用
- DetailsPanel 的视频 6 帧循环预览基于视频 duration 均匀采样，不应只集中在开头
- PDF 第一页缩略图 warmup 可用，开发模式下 `/debug/thumbnails/warmup` 能看到 `pdf_render_mode=subprocess-v1`
- DetailsPanel 的视频 6 帧循环预览只在视频详情中出现
- 手动放入 Games 的 `.exe` 显示现有 exe icon，archive / rom / iso / shortcut 显示稳定 fallback
- Settings 中切换 English / 简体中文后，Sidebar、Shell、当前页面和 DetailsPanel 文案即时刷新
- Light / Dark 主题切换后，Home / Search / Files / Recent 没有明显浅色残留或不可读文字
- Tools 页面可访问，video_merge 工具可正常创建 run（需要 indexed video files 或 external paths）
- 关闭应用后 backend 子进程退出；再次启动不端口冲突

## 7. 已知限制

- 当前仍是 beta，不是正式版发行。
- backend 是随桌面包启动的本地子进程，不安装为系统服务。
- 开发模式下如果直接关闭 PowerShell 窗口，旧 backend 进程可能残留；应优先通过 `Ctrl+C` 正常停止，并用 debug runtime 指纹确认当前进程。
- FFmpeg 来自随包携带的第三方二进制，发布时需要保留对应许可说明。
- pypdfium2 / PDFium 用于 PDF 第一页缩略图，随二进制分发时需要保留 pypdfium2、PDFium 及其依赖的许可说明。
- 视频多帧预览只在 DetailsPanel 中出现，不在列表、hover 或卡片中出现；当前保持 6 帧并使用 duration-aware 采样。
- Collections 仍是 saved retrieval surface，不是规则引擎。
- Games status 仍是 Games 专属语义，不是全站统一 status。
- Games 不是游戏平台或启动器，不联网抓取封面。
- Documents 不是阅读器、Office 编辑器或文档管理平台；`/library/books` 是兼容命名。
- Software 不是安装管理器。
- Archive 通过 Files Archives quick filter 找到，不新增 Archives 独立页面。
- 当前不支持 AI 自动标签、OCR、推荐系统或自动规则。

## 8. 反馈回收建议

外部反馈建议按下面优先级记录：

- P0：无法启动、关键页面崩溃、主路径无法完成、backend 子进程残留导致无法重启
- P1：主路径明显卡住、文案误导严重、桌面动作不可用、视频预览在常见视频上异常
- P2：可带入 beta 的体验瑕疵、低可见文案或视觉问题
- P3：未来功能期待或非当前范围建议

反馈时优先记录：

- 用户使用的 Windows 版本
- 是否从一体化安装包启动
- source 类型与样本文件类型
- 具体在哪个页面、哪个动作出现问题
- 是否能稳定复现

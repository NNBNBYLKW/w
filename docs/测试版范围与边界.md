# 测试版范围与边界

## 1. 文档目的

本文件是当前项目在**测试版阶段**的范围边界文档。

它只回答 4 个问题：

- 当前测试版必须包含什么
- 当前测试版明确不做什么
- 当前对外应如何定义产品
- 当前推进时应优先坚持什么原则

本文件不是新的功能池，也不是新一轮产品规划。

## 2. 当前测试版目标

当前测试版阶段的目标不是继续横向扩线，而是：

> 把已经完成的垂类库能力与组织层能力，收口成一个边界清楚、体验一致、可演示、可验证的测试版。

这一步的重点是：

- 统一
- 收边界
- 打磨主路径
- 减少误导
- 为真实验证做准备

最近这轮前端 UI 调整也属于这一步：它们是在收口三栏壳层、导航 / details 控制和状态表达，不是继续扩新的能力线。

## 3. 测试版范围内的必须项

### 3.1 主导航与基础入口

- Home
- Recent
- Media
- Games
- Books
- Software
- Tags
- Collections
- Settings

### 3.2 四条垂类库主线

#### Media

- 浏览
- 最小筛选
- 轻量整理
- 与 details / tags / collections / recent 的回流

#### Games

- 游戏识别子集
- 库式浏览
- 打开
- 轻量状态
- 再找回闭环

#### Books

- 书库入口
- 信息表达增强
- 与 recent / tags / collections / details 的最小回流

#### Software

- 软件库入口
- 软件识别与信息表达
- 与 recent / tags / collections / details 的最小回流

### 3.3 组织层必须项

- batch tags
- batch color tags
- favorite
- rating
- collections 的保存与最小编辑
- recent imports / tagged / color-tagged
- shared details 中的共通写入与再找回动作

### 3.4 共通体验必须项

- single click -> details
- double click -> desktop open
- browser graceful downgrade
- loading / empty / no-result 基本成立
- shared details 继续是唯一详情中心
- 统一三栏工作台壳层表达
- 左侧导航可展开 / 收起
- details 可显示 / 隐藏，隐藏后中间区补位
- 顶部状态与控制保持轻量、克制并服务主路径

## 4. 当前明确后置的内容

以下内容不属于当前测试版必须项，应后置处理：

### 4.1 垂类深水区能力

- Media 的更深预览体系
- Games 的 launcher / 平台账号 / 时长 / 成就
- Books 的阅读器 / 阅读进度 / 批注
- Software 的安装管理 / 卸载 / 版本中心 / 包管理器

### 4.2 组织层扩展能力

- favorite / rating 跨全站筛选
- 全站统一 status 系统
- batch favorite / rating
- 复杂跨页批处理工作台
- smart collections 规则引擎化
- 更完整的 recent family 时间线化

### 4.3 平台化与智能化能力

- AI 自动标签
- OCR / 语义识别
- 推荐系统
- Dashboard 统计中心
- 自动规则引擎
- 统一对象中台

## 5. 当前明确不做

为了防止测试版继续膨胀，当前阶段明确不做：

- 不新增新的垂类库
- 不重写底层架构
- 不新建统一对象中台
- 不引入重外部 metadata 依赖
- 不做 AI / 推荐 / 自动规则
- 不把任一 subset surface 推向独立平台化
- 不为了“更完整”而加入大而空的新页面
- 不把当前 UI 收口误写成新的窗口系统、菜单系统或平台壳层能力

## 6. 当前对外产品定义

当前测试版对外的最简定义应保持为：

> 一个 local-first 的本地资产工作台测试版。  
> 它围绕 Media / Games / Books / Software 四类本地资产，提供浏览、轻量整理、再找回与统一详情能力。

当前表达中应避免把它误导成：

- 通用文件管理器替代品
- 游戏平台 / 启动器
- 电子书阅读平台
- 软件管理平台
- AI 资产中台

## 7. 当前默认演示路径

建议验证与演示时优先使用以下主路径：

### 路径 A：Recent -> 整理 -> 再找回

1. 打开 Recent Imports
2. 进入 batch mode
3. 批量添加标签 / 颜色标签
4. 回到对应 subset
5. 通过 tags / collections 再找到目标

### 路径 B：Media 主路径

1. 进入 Media
2. 浏览 / 筛选
3. 查看 details
4. 做整理动作
5. 通过 collections / recent 再找回

### 路径 C：Games 主路径

1. 进入 Games
2. 浏览
3. 查看 details
4. 打开游戏条目
5. 设置轻量 status
6. 再按 status 或 tag 找回

### 路径 D：Books / Software 主路径

1. 进入 Books 或 Software
2. 浏览与确认条目语义
3. 查看 details
4. 写入 favorite / rating / tag / collection
5. 通过 recent / collections 再找回

## 8. 当前阶段开发原则

测试版阶段默认遵守这些原则：

1. 先收边界，不先加功能
2. 先统一体验，不先做长尾
3. 先打磨主路径，不先做边缘页
4. 先准备验证，不先做“看上去高级”的能力
5. 任何新增需求先判断是否应后置

当前导航展开 / 收起、details 显隐补位、顶部状态图标化和滚动体验收口，都应按“体验统一”理解，而不是按“新增功能”理解。

## 9. 当前结论

从测试版阶段开始，项目的主节奏应固定为：

> 范围冻结 -> 体验统一 -> 外部验证 -> 问题收敛 -> 稳定性与正式版收口

如果某项工作会把项目重新带回“继续扩功能”的节奏，应优先视为边界风险。

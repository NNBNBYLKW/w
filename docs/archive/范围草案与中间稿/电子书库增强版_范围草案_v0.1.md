# 电子书库增强版（范围草案）v0.1
> 用途：用于定义“阶段三-A：电子书库增强版”的范围、边界、批次拆分与 Codex 执行约束。

## 1. 文档目的

本文档用于定义后续阶段中的“阶段三-A：电子书库增强版”范围。

本草案不是重定义产品，而是在当前已成立的本地资产工作台骨架上，提升 Books 的浏览、检索、组织与再次找回体验。

本阶段默认继续以当前系统真实能力与 current-state 边界为前提，不把电子书库推进成阅读平台。

## 2. 当前起点

当前项目已经是一个 Windows local-first 本地资产工作台，建立在真实文件系统之上，以 files 为统一索引对象。

Books 当前已经作为真实 subset surface 存在，本质是 active indexed .epub / .pdf 文件的识别子集入口。

Books 目前仍复用 shared details、tags、color tags、collections、Search 与 open actions，而不是独立书籍数据库或阅读系统。

## 3. 本阶段目标

把当前 Books 从“ebook 文件子集列表”增强为一个更像“电子书库入口”的轻量测试版。

用户应能够更自然地依靠封面、标题、作者、格式、标签等信息定位书籍，而不是只依赖路径和文件名。

在不平台化的前提下，让书库具备更明确的浏览感、组织感和再次找回能力。

## 4. 本阶段明确不做什么

不做阅读器。

不做阅读进度系统、批注系统、书摘系统。

不做在线元数据重依赖方案，不把联网抓取当作本阶段成立前提。

不做独立书籍对象中台，不脱离 files 建立新的正式事实层。

## 5. 架构与边界前提

indexed files 仍是底座；Books 仍是 subset surface。

shared details 仍是共同详情与动作面，不分叉出独立书籍详情系统。

Search / tags / color tags / collections 仍是横向复用组织层。

桌面 open actions 仍通过现有薄壳承接；默认动作仍是打开原文件或打开所在目录。

不引入“阅读平台化”“书库平台化”的新产品主轴。

## 6. 范围总览

本阶段建议分三批次推进，先做展示密度，再做组织入口，最后收口“详情—打开—再次找回”的闭环。

## 7. 批次 A：书库展示密度增强

7.1 目标

让用户首先在视觉上感知 Books 已经是“书库入口”，而不是单纯的 ebook 扩展名列表。

7.2 包含内容

增强 Books 页面中的封面优先级与卡片/列表信息层级。

在可稳定提取时展示标题、作者、格式等字段；提取失败时保守回退到文件名与扩展名。

优化空状态、加载状态、封面缺失占位态。

保持 shared details 作为共通详情容器，不新增独立书籍详情体系。

7.3 不包含内容

不做封面精修工具。

不做目录提取、章节导航、正文阅读体验。

不因为个别格式差异引入新的复杂解析平台。

7.4 验收口径

用户进入 Books 时首先看到的是更像书库的展示，而不是文件清单。

封面、标题、作者等信息在“可提取时”明确改善定位体验。

封面缺失时的回退表现不显得残缺或破坏整体观感。

## 8. 批次 B：组织与检索入口增强

8.1 目标

让用户能通过封面、标题、作者、格式、标签等入口更自然地找书，而不是主要退回 Search/Files。

8.2 包含内容

强化 Books 页中的标签、格式、基础筛选与排序入口。

在现有体系下增强作者/标题等字段在页面与 details 中的展示优先级。

让 Books 与 Search / Collections / Tags 的关系更清晰，支持从书库入口回流到横向组织层。

8.3 后端允许的最小补强

允许对 Books 列表字段做轻量补充或稳定性增强。

允许对现有详情字段增加最小书籍展示字段。

不引入新的正式书籍对象层；不把联网 metadata 作为必要依赖。

8.4 不包含内容

不做复杂作者库 / 系列库中台。

不做推荐系统、阅读偏好系统、云同步书架。

8.5 验收口径

用户能在 Books 内通过更自然的入口缩小范围并确认目标。

Books 的检索体验明显比当前 subset v1 更接近“轻量书库”。

## 9. 批次 C：书库闭环收口

9.1 目标

把 Books 从“可浏览”推进到“可归类、可再次找回”的轻量书库入口。

9.2 包含内容

强化 details 中的标签、颜色标签、打开文件、打开目录等动作与 Books 的协同。

强化 Books 与 Collections、Tags、Search 的回流闭环。

在不进入批注/阅读进度体系的前提下，提升“归类—再次找到”的自然度。

9.3 不包含内容

不做阅读进度、已读/在读生态。

不做笔记、摘录、批注、同步系统。

9.4 验收口径

用户能够从 Books 进入详情、归类并再次从 Tags/Collections/Search 找回目标书籍。

Books 已经具备“轻量书库入口”的使用闭环，而不仅是展示层增强。

## 10. 页面与组件影响范围

主要影响页面：/books、/search（仅限书籍结果表现相关部分）、/tags、/collections（仅限书籍回流相关部分）。

主要影响 feature / 组件：BooksPage 相关 feature、DetailsPanelFeature、FilterBarFeature、TaggingFeature、FileCard / MediaCard 的书籍变体表现（如已有）、空状态 / 骨架态 / 无结果态组件。

默认不新增独立书籍详情体系，不新增书籍私有组织系统。

## 11. 后端与数据层边界

默认复用 files 统一索引对象、现有 Books subset route、现有 GET /files/{id} 详情主链。

允许的补强：Books 列表轻量字段增强、展示字段稳定性增强、基础过滤/排序能力补充。

明确禁止：新建独立书籍中心表替代 files；引入复杂异步解析流水线；把外部元数据平台拉成事实层。

## 12. 交互原则

单击：选中并在右侧 details 中查看。

双击：打开原文件。

筛选即时生效；标签和颜色标签尽量在当前页就地完成。

details 侧栏仍是高频整理中心，不退化成纯说明面板。

## 13. 进入前提

Books route 已存在，shared details 主链可用，tags / color tags / collections 可复用，桌面 open actions 仍由现有薄壳承接。

本阶段不要求先重写架构或先建独立书籍模型。

## 14. 退出条件

Books 已明显超出当前 subset v1 水平，形成轻量书库气质。

浏览、检索、详情、归类、再次找回几条链能够自然连通。

未滑向阅读平台、在线书架平台或重对象中台方向。

## 15. 给 Codex 的执行约束

不改写项目当前定位，不把 Books 做成阅读平台。

不分叉 shared details 体系，不脱离 files 建立新事实层。

能复用现有 feature / shared components 的，优先复用。

前端优先做“体验增强与链路打顺”，不是为了未来抽象过度设计。

## 16. 本阶段后的自然下一步

电子书库增强版完成后，顺序上应继续进入《软件库增强版（范围草案）》；而不是提前跳向阶段六的稳定性 / 性能总收口。

## 17. 当前待拍板事项

Books 首版是否显式区分 PDF / EPUB 两个子范围。

封面缺失时的默认回退卡片样式如何定义。

作者 / 标题字段的提取失败时，页面上如何回退展示。

是否允许做少量 backend 字段增强。

阶段三-A 是否拆成 3 个独立 Codex 任务执行，还是 1 个整阶段任务执行。

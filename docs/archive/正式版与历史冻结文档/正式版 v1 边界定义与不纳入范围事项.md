# 正式版 v1 边界定义与不纳入范围事项

## 1. 文档目的

本文档用于把当前正式版 v1 的产品边界固定为**当前代码与当前验收口径已经成立的范围**。

它的作用是：

- 说明当前 frozen v1 到底包含什么
- 说明哪些事项明确不属于当前交付承诺
- 防止把未来愿景或历史规划反推成“当前正式版漏做项”

本文档是当前状态文档，不是路线图，也不承担扩 scope 作用。

---

## 2. 当前 frozen v1 的准确定义

当前正式版 v1 应被表述为：

> **一个 local-first 的本地资产工作台基础版，支持 source 管理、扫描索引、多入口检索与浏览、基础 metadata、图像缩略图与预览、标签与颜色标签组织，以及可保存的 collections 入口。**

这个定义强调三点：

- 当前系统是本地文件资产工作台，而不是 Explorer replacement
- 当前系统已经有真实可用的主链，而不是占位式 demo
- 当前系统是基础版，不等于未来所有方向都已经进入 v1

---

## 3. 当前 v1 明确包含的能力

### 3.1 Source 与扫描链

当前 v1 包含：

- source 创建
- source root 校验与重复/重叠保护
- 手动触发 scan
- delete-sync
- same-source active scan 冲突保护
- source 级最近失败反馈

### 3.2 多入口检索与浏览

当前 v1 包含以下真实入口：

- Search
- Files
- Media Library
- Recent Imports
- Tags
- Collections

这些入口都服务于当前 active indexed files，而不是快照式或演示式数据面。

### 3.3 组织层

当前 v1 包含：

- normal tags attach / remove
- color tag set / clear
- Search / Files 的 tag 与 color-tag 检索增强
- Collections 作为长期可复用检索入口

### 3.4 内容增强层

当前 v1 包含：

- image metadata baseline
- image thumbnail
- image preview

这些能力让当前工作台具备了最小但真实的内容可见性。

### 3.5 详情与动作

当前 v1 包含：

- shared `DetailsPanelFeature`
- file details
- metadata 展示
- tags / color tags 展示与操作
- `Open file`
- `Open containing folder`

---

## 4. 当前 v1 明确不纳入的事项

以下事项不属于当前正式版 v1 的交付承诺，不应被误记为 bug：

- 完整 task platform、scan history、retry system、runtime center
- video poster extraction、video hover/play、document preview、富媒体预览系统
- video/document metadata 的正式激活
- advanced query builder、OR / NOT、多条件表达式
- smart rules / automation engine
- game / book / software 垂类库
- AI / semantic / OCR / embeddings
- batch actions
- cloud sync、multi-device、auth
- dashboard 化 Home
- preferences 化 Settings
- Explorer replacement / 目录树型 Files 页面

---

## 5. 常见误解与纠偏

### 5.1 有 Collections，不等于已经有规则引擎

当前 Collections 只是保存最小结构化条件并重新进入实时结果视图，不是 automation engine。

### 5.2 有 image metadata / preview，不等于已经有完整媒体系统

当前正式成立的是 image-first enrichment，而不是所有媒体类型的 preview 与 metadata 体系。

### 5.3 有 scan/runtime hardening，不等于已经有 task center

当前只补齐了最小运行时收口，不包含任务历史、队列、重试或独立 runtime 页面。

### 5.4 Search / Files 有过滤，不等于已经有高级查询器

当前过滤语义仍刻意保持为受控的最小表达，而不是可组合的高级规则系统。

## 7. 结论

当前版本可以在上述边界内冻结为正式版 v1。

判断标准不是“未来愿景是否全部实现”，而是：

- 当前主链是否已经成立
- 当前产品定位是否清楚
- 范围外事项是否被明确挡住

按当前代码、测试与配套文档，这个边界已经成立。

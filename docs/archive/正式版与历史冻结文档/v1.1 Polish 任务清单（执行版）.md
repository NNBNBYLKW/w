# v1.1 Polish 任务清单（执行版）

## 1. 文档目的

本文档用于把正式版 v1 之后的第一轮 polish 工作限制在**当前边界内的小幅完成度提升**。

它不是：

- 新功能路线图
- 下一阶段大功能池
- 借 polish 重新扩大 v1 边界的入口

---

## 2. 当前 v1.1 定位

当前 v1.1 定义为：

> **对正式版 v1 基线做小幅、可感知、但不改产品语义的 polish / follow-up。**

本文档当前作为已落地 v1.1 polish 的 follow-up record 使用，而不是新的范围规划入口。

它的重点是：

- 让现有主链更像正式软件
- 让已有入口更自然
- 让文档与当前状态更容易交接

---

## 3. 当前批次记录

### Batch 1：Source picker + source create polish

当前已拆成更窄子批次推进。

#### Sub-batch 1：desktop folder picker integration

本子批次只覆盖：

- desktop shell 的最小 folder picker bridge
- source create path 输入框的最小 picker 按钮接入
- 桌面模式选择目录后回填 path
- 浏览器模式继续保留手输路径并优雅降级

本子批次不覆盖：

- source wording cleanup
- source create error-message rewrite
- first-scan guidance
- backend source APIs
- runtime / scan 语义调整

#### Sub-batch 2：source create wording / error polish

本子批次只覆盖：

- source path 的短 inline helper
- source-create 局部错误文案收口
- source-create 区域内的成功 / next-step 提示

本子批次不覆盖：

- backend error-code changes
- source create success/error semantics
- first-scan guidance redesign
- runtime / scan 语义调整

#### Sub-batch 3：first-scan guidance and empty-state polish

本子批次只覆盖：

- no-sources empty state 的本地文案收口
- source-create 成功后的下一步提示收口
- source row 现有 action/status 语境中的首次 scan 提示

本子批次不覆盖：

- source picker 变化
- source create error remapping 扩张
- backend/runtime/task 语义调整
- task-center-like UI

### Batch 3：Docs 与 bootstrap 与 packaging 收口

当前只覆盖：

- canonical/current-state docs clarification
- concise operational entrypoint clarification
- bootstrap reality clarification
- current validation wording clarification

当前不覆盖：

- packaging automation implementation
- CI / installer redesign
- backend / frontend / desktop implementation changes

---

## 4. 当前仍保持的约束

v1.1 仍然不进入以下方向：

- backend 行为扩张
- scan/runtime 平台改造
- retrieval 逻辑变更
- desktop 深系统集成
- 新页面 / 新一级产品主题

# v1.1 Batch 1：Source picker + source create polish（执行版）

## 1. 文档目的

本文档用于记录 v1.1 Batch 1 的当前收口边界。

它当前是已落地切片的 execution/current-state record，不承担 startup/build/run 指南角色。

当前 Batch 1 只聚焦：

> **Source picker + source create polish**

它不是 source 系统重做，也不是 runtime 扩张批次。

---

## 2. 当前 Batch 1 的拆分

### Sub-batch 1：desktop folder picker integration

当前执行范围只包括：

- desktop preload bridge 支持真实目录选择
- source create 表单接入最小 picker 按钮
- 选择目录后回填现有 path 输入框
- 取消选择时保持当前输入不变
- 浏览器模式下按钮安全降级，手输路径继续可用

当前明确不包括：

- source create wording cleanup
- source create error-message polish
- first-scan guidance
- backend source API 语义变化
- scan/runtime 逻辑变化

### Sub-batch 2：source create wording / error polish

当前执行范围只包括：

- source path 的短 inline helper
- source-create 局部错误文案映射
- source-create 区域内的成功 / next-step 提示

当前明确不包括：

- backend/source validation semantics 变化
- backend error-code 变化
- first-scan guidance redesign
- scan/runtime 逻辑变化

### Sub-batch 3：first-scan guidance and empty-state polish

当前执行范围只包括：

- no-sources 空态的本地文案收口
- source-create 成功后的下一步提示收口
- source row 现有 action/status 语境中的首次 scan 提示

当前明确不包括：

- source picker 变化
- source create error remapping 扩张
- backend/runtime/task 语义变化
- task UI 扩张

---

## 3. 当前实现口径

### Desktop shell mode

- 提供一个最小 `Choose folder` 动作
- 调起系统目录选择器
- 成功选择后返回单个目录路径
- 取消时返回 `null`

### Browser-only mode

- 不发明伪 picker
- 不破坏现有 manual path entry
- picker 按钮仅做不可用降级

### Source create wording / error polish

- path helper 只用一条短 inline 文案
- create errors 只在本地 source-create surface 内收口
- create success 只在 source-create 区域内提示下一步建议执行 scan
- 不加全局 toast，不加 page-wide banner，不触发自动 scan

### First-scan guidance and empty-state polish

- no-sources 空态只用一条简短文案说明先添加 source，再执行 scan
- post-create note 只负责确认 source 已添加并指出下一步
- per-source hint 只在真实 source row 的 action/status 语境里解释 scan 会索引该目录中的文件
- 不重复提示，不引入 task 语言，不扩张 scan controls

---

## 4. 当前 Batch 1 收口结果

当前 Batch 1 已收口的结果包括：

- 桌面壳中点击 picker 能打开系统目录选择器
- 选中目录后 source path 输入框被正确回填
- 取消选择时当前输入值保持不变
- source-create helper 文案更清楚
- source-create 局部错误文案更容易理解
- source-create 成功后会在本地区域提示下一步运行 scan
- no-sources 空态和首次 scan 提示更容易理解
- 使用选中的路径创建 source 成功
- source 创建后的 scan 行为无回退
- 浏览器模式下手输路径仍正常工作
- frontend build 与 desktop build 都通过

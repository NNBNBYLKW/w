# 测试版当前状态总览

## 1. 当前结论

这个仓库当前已经进入 **测试版阶段**。

项目应被理解为一个 **Windows local-first 本地资产工作台**：

- 真实文件系统是事实层
- `files` 与相关元数据表是组织层
- `shared details` 是统一高频操作中心
- `Media / Games / Books / Software` 是受控 subset surfaces
- `Recent / Tags / Collections` 是组织与再找回入口

当前推进重点已经从“继续扩功能”切换为：

> 范围冻结 -> 体验统一 -> 验证准备 -> 问题收敛

当前前端也已经更明确地收口成统一工作台，而不是一组相对松散的页面集合。

## 2. 当前项目是什么

当前产品是：

- 一个建立在本地文件系统之上的 local-first 工作台
- 一个围绕已索引文件进行浏览、检索、标记、再找回与打开的系统
- 一个共享同一套 details、tags、color tags、collections、favorite/rating 与 open actions 的统一产品

当前产品不是：

- Windows Explorer replacement
- 游戏平台或启动器
- 电子书阅读器
- 软件安装管理器 / 包管理器
- AI 平台 / 推荐平台 / 自动规则系统
- 独立对象中台驱动的多产品集合

## 3. 当前已完成的主线能力

### 垂类库

- `Media`
  - 图片 / 视频浏览
  - 最小筛选
  - 与 details / tags / collections / recent 的回流
- `Games`
  - 游戏入口文件识别
  - 库式浏览
  - 轻量状态
  - 与 details / tags / collections / recent 的回流
- `Books`
  - 书库入口
  - 书籍信息表达增强
  - 与 details / tags / collections / recent 的回流
- `Software`
  - 软件入口
  - 软件识别与信息表达
  - 与 details / tags / collections / recent 的回流

### 组织层

- normal tags
- color tags
- batch organize
  - batch tags
  - batch color tags
- favorite
- rating
- collections
  - saved retrieval
  - minimal edit
- recent family
  - imports
  - tagged
  - color-tagged

### 共通能力

- single click -> select + show details
- double click -> open file
- browser mode graceful downgrade
- shared details 作为唯一详情中心
- 从 details / tags / collections / recent 回到各 subset 的最小闭环

## 4. 当前主要页面面

当前产品已经形成一套稳定的主页面组合：

- `Home`
- `Search`
- `Files`
- `Recent`
- `Media`
- `Games`
- `Books`
- `Software`
- `Tags`
- `Collections`
- `Settings`

其中：

- `Search` 和 `Files` 是通用面
- `Media / Games / Books / Software` 是垂类 subset surfaces
- `Recent / Tags / Collections` 是组织与 retrieval surfaces

这些页面都没有脱离统一的 indexed-files 主语义。

### 当前工作台壳层形态

当前前端界面已经更明确地形成：

- 左侧导航区
- 中间浏览 / 检索区
- 右侧 shared details 区

并且当前交互形态已经进一步统一：

- 左侧导航支持展开 / 收起
- 右侧 details 支持显示 / 隐藏，隐藏后中间区补位
- 顶部区域使用更克制的 backend 图标状态提示
- details 显隐按钮与 sidebar toggle 使用更接近的控制样式
- 导航图标继续统一走 SVG 文件资源 + 组件封装 + `currentColor`

这些变化说明当前工作台的 UI 形态已经更稳定，但它们仍然属于测试版体验收口，不应被理解成新的产品能力批次。

## 5. 当前共享中心

`DetailsPanelFeature` 仍然是整个工作台的统一详情中心。

它当前承接的高频行为包括：

- 基本文件信息
- metadata
- tags
- color tag
- favorite / rating
- game status
- book info / software info 等轻量垂类确认区块
- open actions
- refind CTA

当前没有分叉出 page-specific details system。

因此，当前前端的主语义仍然是：不同页面共享同一套 shell、同一 details 入口、同一状态表达，而不是各页各自长成独立产品。

## 6. 当前边界

虽然当前已经有四条垂类库和一层轻量组织能力，但边界仍然清楚：

- `Collections` 仍是 saved retrieval surface，不是 smart-rules platform
- `Recent family` 仍是“方便再次找到”的 retrieval family，不是事件分析中心
- `Games status` 仍是 Games 专属语义，不是全站统一 status
- `Books` 不是阅读平台
- `Software` 不是安装管理平台
- `Media` 不是更宽的资产制作平台

## 7. 当前阶段应该怎么推进

当前阶段不应再沿着“再补一个新能力批次”的方式推进。

更合适的节奏是：

1. 维持测试版范围边界
2. 收口体验差异
3. 做真实验证
4. 根据验证结果做 P0 / P1 问题修正
5. 再进入稳定性、性能与正式版收口

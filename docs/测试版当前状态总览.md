# 测试版当前状态总览

## 1. 当前结论

这个仓库当前已经进入 **Release Candidate v0.3.0** 阶段。

项目应被理解为一个 **Windows local-first 本地资产工作台**：

- 真实文件系统是事实层
- `files` 与相关元数据表是组织层
- `shared details` 是统一高频操作中心
- `Media / Games / Books / Software` 是受控 subset surfaces
- `Recent / Tags / Collections` 是组织与再找回入口

当前推进重点已经完成从“继续扩功能”到范围冻结 -> 体验统一 -> 验证准备 -> 问题收敛的完整切换。

项目已经完成 7 个阶段（Phase 9-16）、168 个任务的实施，包括 AI 识别、回收站/恢复、游戏启动器、视频悬浮预览、智能去重等深水区能力。

当前前端也已经更明确地收口成统一工作台，而不是一组相对松散的页面集合。

当前 Windows 交付形态也已经完成从技术预览包到一体化 beta 包的升级：

- Electron packaged mode 会自动启动随包携带的 backend executable
- backend 运行数据写入 Electron `userData` 下的 `backend-data/`
- 视频缩略图与 DetailsPanel 6 帧预览默认使用随包携带的 `ffmpeg.exe`
- 旧的需手动 backend 的技术 beta 包仅作为归档回退，不再作为推荐交付形态

## 2. 当前项目是什么

当前产品是：

- 一个建立在本地文件系统之上的 local-first 工作台
- 一个围绕已索引文件进行浏览、检索、标记、再找回与打开的系统
- 一个共享同一套 details、tags、color tags、collections、favorite/rating 与 open actions 的统一产品

当前产品不是：

- Windows Explorer replacement
- 游戏平台或启动器
- 电子书阅读器
- 软件安装管理器 / 包管理器
- AI 平台 / 推荐平台 / 自动规则系统
- 独立对象中台驱动的多产品集合

## 4. 当前主要页面面

当前产品已经形成一套稳定的主页面组合：

- `Home`
- `Search`
- `Files`
- `Recent`
- `Media`
- `Games`
- `Books`
- `Software`
- `Tags`
- `Collections`
- `Settings`

其中：

- `Search` 和 `Files` 是通用面
- `Media / Games / Books / Software` 是垂类 subset surfaces
- `Recent / Tags / Collections` 是组织与 retrieval surfaces

这些页面都没有脱离统一的 indexed-files 主语义。

### 当前工作台壳层形态

当前前端界面已经更明确地形成：

- 左侧导航区
- 中间浏览 / 检索区
- 右侧 shared details 区

并且当前交互形态已经进一步统一：

- 左侧导航支持展开 / 收起
- 右侧 details 支持显示 / 隐藏，隐藏后中间区补位
- 顶部区域使用更克制的 backend 图标状态提示
- details 显隐按钮与 sidebar toggle 使用更接近的控制样式
- 导航图标继续统一走 SVG 文件资源 + 组件封装 + `currentColor`

这些变化说明当前工作台的 UI 形态已经更稳定，当前属于 Release Candidate v0.3.0 收口阶段。

## 5. 当前共享中心

`DetailsPanelFeature` 仍然是整个工作台的统一详情中心。

它当前承接的高频行为包括：

- 基本文件信息
- metadata
- tags
- color tag
- favorite / rating
- game status
- book info / software info 等轻量垂类确认区块
- open actions
- refind CTA

当前没有分叉出 page-specific details system。

当前视频预览也遵循这个边界：列表页只消费统一 thumbnail；6 帧静态循环预览只在 shared details 中对当前选中的视频出现，失败时回退到单帧或占位，不影响组织主路径。

因此，当前前端的主语义仍然是：不同页面共享同一套 shell、同一 details 入口、同一状态表达，而不是各页各自长成独立产品。

## 6. 当前边界

虽然当前已经有四条垂类库和一层轻量组织能力，但边界仍然清楚：

- `Collections` 仍是 saved retrieval surface，不是 smart-rules platform
- `Recent family` 仍是“方便再次找到”的 retrieval family，不是事件分析中心
- `Games status` 仍是 Games 专属语义，不是全站统一 status
- `Books` 不是阅读平台
- `Software` 不是安装管理平台
- `Media` 不是更宽的资产制作平台

## 7. 当前阶段总结 (v0.3.0 RC)

项目已完成从 Phase 9 到 Phase 16 的全部 7 个阶段、168 个任务。

当前状态：

1. 范围边界已稳定 —— 四条垂类库 + 组织层 + 通用能力已就绪
2. 深水区能力已完成 —— AI 分类建议、回收站/恢复、游戏启动器、视频悬浮预览、智能去重等
3. BrowseV2 已完成 36 项 UX 增强 —— 包括多选、键盘导航、上下文菜单、拖放、视图模式切换等
4. 正在收尾残余体验差异，进入正式版发布准备
5. 不再以”再补一个新能力批次”的方式推进
